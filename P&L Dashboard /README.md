## 💡 О проекте

Компания работает в **5 странах** (A–E), привлекая клиентов через платный трафик. Бизнес-модель: трафик закупается на **всех** клиентов, но доход получается только с **подтверждённых**.

Проект решает три аналитические задачи:

1. **P&L по странам** — оценка прибыльности каждого рынка с учётом неподтверждённых конверсий
2. **Cash-Flow на 60 дней** — моделирование кассовых разрывов при задержке выплат (30–40 дней)
3. **Сценарный анализ** — стресс-тест EBITDA при ухудшении ключевых операционных параметров

---

## 🧮 Финансовая модель

### P&L

| Метрика | Формула |
|---|---|
| Подтверждённые клиенты | `Все клиенты × (1 − % неподтверждённых)` |
| Чистый доход | `Подтверждённые × Доход за клиента` |
| Переменные расходы | `Все клиенты × Opex за клиента` |
| Валовая прибыль | `Чистый доход − Переменные расходы` |
| EBITDA | `Валовая прибыль − (Команда + IT + Банк 3.9%)` |

**Фиксированные расходы:** команда — $250K/мес, IT — $80K/мес, банковская комиссия — 3.9% от оборота.

### Cash-Flow

Модель учитывает задержку поступлений:
- **70% дохода** — поступает с задержкой 30 дней
- **30% дохода** — поступает с задержкой 40 дней
- Начальный резерв — $800K

Рассчитываются: день кассового разрыва, необходимый резерв для покрытия, сравнение базового сценария vs рост +30%.

### Сценарный анализ

| Сценарий | Изменение | Эффект |
|---|---|---|
| Сценарий 1 | Неподтверждённые +5 п.п. | ↓ доход |
| Сценарий 2 | Opex ×1.10 | ↓ валовая прибыль |
| Сценарий 3 | Доход за клиента ×0.93 (-7%) | ↓ EBITDA |

## 📈 Power BI Dashboard

**Страниц:** 1 | **Таблиц в модели:** 3 (`countries`, `cashflow`, `summary`)

### Визуализации

| Чарт | Тип | Вопрос |
|---|---|---|
| Где компания зарабатывает? | Столбчатая диаграмма | Gross Profit по странам |
| Где компания теряет маржу? | Столбчатая диаграмма | Gross Margin по странам |
| Где риск перегрева? | Таблица | Unconfirmed Rate + Risk по странам |
| Что произойдёт при масштабировании? | Линейный график | Balance_Base vs Balance_Growth30 (60 дней) |
| Запас прочности по ликвидности | KPI-карточки | Gap Day, Required Reserve (база и +30%) |

### Поля модели данных

**`countries`:** `Country`, `Clients`, `Gross_Profit`, `Gross_Margin`, `Unconfirmed_rate`, `Risk`

**`cashflow`:** `Day`, `Balance_Base`, `Balance_Growth30`

**`summary`:** `Metric`, `Value`

## 📌 Ключевые инсайты, заложенные в модель

- **Страна E** — наибольший процент неподтверждённых (18%), что делает её самым рискованным рынком при масштабировании
- **Кассовый разрыв** наступает в первые 30 дней из-за задержки выплат — при росте +30% резерв нужно увеличивать непропорционально
- **Сценарий "Opex +10%"** — наиболее болезненный для EBITDA, так как opex начисляется на всех клиентов, включая неподтверждённых
