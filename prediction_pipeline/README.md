# 🎯 EGE Score Prediction Pipeline

> ML-пайплайн для прогнозирования баллов ЕГЭ на основе результатов пробных экзаменов. Запускается в один клик в Google Colab — без установки, без настройки.


---

## 🔍 Что это и зачем

Репетиторские центры и школы каждый год сталкиваются с одним вопросом: **кто из учеников в зоне риска?** Обычно ответ — интуиция куратора. Этот проект делает то же самое, но на данных.

Ноутбук берёт историю пробных экзаменов, обучает несколько моделей на данных прошлого года и выдаёт прогноз для каждого ученика текущего года: ожидаемый балл, диапазон и цветовую зону риска.

**На выходе куратор получает два файла:**
- `прогноз.xlsx` — таблица по каждому ученику
- `отчёт.pptx` — презентация для руководства

---

## ⚙️ Как работает

Каждый запуск — это полный цикл AutoML на ваших данных:

1. **Парсинг и очистка** — пропуски в пробниках обрабатываются корректно (не писал ≠ ноль)
2. **Перебор моделей** — тестируется до 8 алгоритмов на исторических данных
3. **Выбор победителя** — по двум метрикам: точность числа и точность зоны
4. **Прогноз** — лучшая модель применяется к текущему году

При добавлении новых данных модель автоматически пересматривается.

---

## 📂 Входные данные

Четыре CSV-файла в одной папке с ноутбуком:

| Файл | Содержимое |
|------|------------|
| `годовой_обучение.csv` | Прошлый год, годовой курс |
| `полугодовой_обучение.csv` | Прошлый год, полугодовой курс |
| `годовой_прогноз.csv` | Текущий год, годовой курс |
| `полугодовой_прогноз.csv` | Текущий год, полугодовой курс |

**Колонки файлов обучения:** `ученик`, `пробник_1` ... `пробник_N`, `егэ`

**Колонки файлов прогноза:** `ученик`, `куратор`, `пробник_1` ... `пробник_N`

> Файлы удобнее всего собрать в Google Sheets и скачать через **Файл → Скачать → CSV**.

---

## ▶️ Запуск

```
1. Открыть ноутбук в Google Colab (кнопка выше)
2. Загрузить 4 CSV-файла через иконку папки 📁 слева
3. Среда выполнения → Перезапустить сеанс и выполнить весь код
4. Скачать прогноз.xlsx и отчёт.pptx
```

Никаких зависимостей устанавливать не нужно — всё работает в стандартном окружении Colab.

## 📋 Как читать результаты

| Колонка | Описание |
|---------|----------|
| `имя` | ФИ ученика |
| `куратор` | ФИ куратора |
| `прогноз` | Ожидаемый балл ЕГЭ |
| `от / до` | Вероятный диапазон |
| `зона` | Категория подготовки |

**Зоны:**

| Зона | Порог | Значение |
|------|-------|----------|
| 🔴 Риск | < 64 | Требует внимания куратора |
| 🟡 Средняя | 64–78 | Стандартная траектория |
| 🟢 Сильный | > 78 | Высокий уровень |

> Если ученик написал менее 3 пробников — в колонках `прогноз` и `от/до` будет `—`. Зона при этом всё равно рассчитывается по имеющимся данным.

---

## ⚠️ О точности

Средняя погрешность прогноза — **около 8 баллов**. Модель обучается на данных прошлого года и не учитывает изменения в KIM, состояние ученика в день экзамена и другие внешние факторы.

Прогноз — это **инструмент куратора**, а не замена его суждению. Наибольшая ценность — в зонах риска: они позволяют вовремя сфокусировать внимание на тех, кому нужна помощь.

---

## 🔄 Обновление модели каждый год

```
1. годовой_прогноз.csv      → переименовать в годовой_обучение.csv
   полугодовой_прогноз.csv  → переименовать в полугодовой_обучение.csv

2. Добавить колонку `егэ` с реальными баллами в оба файла

3. Подготовить новые файлы прогноза для следующего года

4. Запустить ноутбук заново
```

Модель сама переобучится на новых данных и выберет лучший алгоритм.
